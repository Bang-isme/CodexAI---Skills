# Output Contracts

Use output contracts to make answers falsifiable and actionable.

## Minimum Contract

Every substantial output should contain:

- chosen path
- why it was chosen over alternatives
- evidence or commands
- risks or open questions
- next step or exit criterion

## Workflow Contract

For process improvements, also include:

- owner or responsible role
- monitoring signal
- rollback or fallback path

## Review Contract

For reviews or audits, prefer:

- finding
- why it matters
- concrete affected boundary
- suggested fix or decision needed

## Implementation Contract

For code work, prefer:

- exact files
- command to verify
- what changed in behavior
- what was deliberately not changed
